# 新しいアプリの追加手順

## 1. カテゴリを決める

- **games(あそび)** — 遊びが主目的。1日の回数制限(共有🎈チケット)の対象。
  各ラウンド開始時に `KidsTickets.useOne()` でゲートし、`false` ならおやすみ画面を出す
  (この配線はテンプレートに入っている)。
- **study(べんきょう)** — 学習が主目的。無制限。チケットには一切触れない。

迷ったら: 学習要素が主なら study。「何回でもやっていい」と親が思えるかで判断する。

## 2. 変数を決める

| 変数 | 規約 |
|---|---|
| `--id` | 英小文字のローマ字1語(例: `pitatto`)。ディレクトリ名と localStorage キー接頭辞になる |
| `--name` | ひらがな中心の短い名前(例: `ぴたっとストップ`)。読めない子向けなのでタイル上は絵文字が主役 |
| `--emoji` | アプリの顔になる絵文字1つ。既存アプリ(🐰🗿🐸🧮🍎🥕)と被らないこと |
| `--desc` | タイルの説明文。ひらがな+絵文字で、分かち書き(例: `はっぱのうえで タッチして とめよう`) |
| `--color` | テーマ色 `#RRGGBB`。淡いパステル系(既存: 空色 #BDE8FF、若草 #DCF0D2 など)。アプリごとに世界観を変えてよい |

## 3. 枠を生成する

```bash
python3 tools/new_app.py --type games --id hoge --name "ほげキャッチ" \
    --emoji 🐙 --desc "おちてくる ほげを キャッチ!" --color "#FFE9D6"
```

これで以下がまとめて行われる(手作業での追随は不要):

- `games/hoge/index.html` をテンプレート(`templates/app.html`)から生成
- ハブ `index.html` のマーカー位置にタイルを挿入
- `sw.js` の ASSETS に2エントリ追加+`CACHE_NAME` を自動インクリメント

## 4. 中身を実装する

生成された `index.html` の `TODO` コメントの箇所にアプリ本体をつくる。守ること:

- **文字に頼らない**: 絵文字・色・音で伝える。文は補助(ひらがな+分かち書き)
- **音**: `KidsSound.tone()` で合成(`docs/reference/shared-modules.md` 参照)。
  音声素材・外部依存は使わない。AudioContext はユーザー操作から触る
- **レベル永続化**: レベル制にするなら `shared/progress.js`(`KidsProgress`)を使う
- **localStorage キー**: `<app-id>-<用途>`(例: `hoge-bests`)。
  追加したら `docs/reference/storage-keys.md` に必ず追記
- **アニメーション**: `prefers-reduced-motion: reduce` の無効化をテンプレートの
  該当ブロックに書き足す

## 5. 確認する

- `python3 tools/check_sw.py` が通ること
- `node tools/smoke.mjs` が通ること(要ローカルサーバー。スクリプト冒頭のコメント参照)
- 4レイアウトで表示崩れがないこと: 縦持ち(〜430px)/ タブレット(600px〜)/
  横2カラム(landscape+640px〜)/ 低い横画面(高さ500px以下)
- オフラインでも動くこと(一度読み込んだあと機内モードでリロード)
