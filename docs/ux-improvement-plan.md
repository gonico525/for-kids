# UX改善計画

対象: こどもひろば全体(2026-07 時点のUXレビューより)。
前提となる設計方針: 対象は4〜6歳の文字が読めない子ども。絵文字・色・音で伝える。外部依存なし・オフライン動作。

優先度順に3件。いずれも未着手(このドキュメントが計画)。

---

## 1. ヌマーカステン: 問題文の音声読み上げ

### 課題

クイズの問題文(「ぜんぶで 13こ。かくれているのは なんこ?」など)が文字のみで、
対象年齢の子どもは読めない。現状は大人が隣で読み上げる前提になっており、
「文字が読めなくても遊べる」という本アプリ群の方針から外れている。

### 方針

ブラウザ内蔵の音声合成(Web Speech API / `speechSynthesis`)で問題文を読み上げる。
外部素材・依存なしの方針を保てる。

- 出題時(`newQuestion`)に問題文を自動で読み上げる
- 問題文バー(`#qtext`)をタップしたらもう一度読み上げる(聞き逃し対策)
- 正解・不正解のメッセージも同様に読み上げる
- 🔊/🔇 は `shared/sound.js`(`KidsSound.enabled()`)の共通設定に従う

### 実装メモ

- `SpeechSynthesisUtterance` に `lang='ja-JP'` を設定。日本語ボイスは
  `speechSynthesis.getVoices()` から `lang` が `ja` のものを選ぶ
  (iOSは初回 `getVoices()` が空のことがあるため `voiceschanged` を待つ)
- 連続出題時は `speechSynthesis.cancel()` で前の読み上げを止めてから話す
- 読み上げ処理は `shared/` に切り出す(例: `shared/speech.js`、`window.KidsSpeech`)と
  今後のべんきょうアプリでも使い回せる
- `rate` はややゆっくり(0.9前後)が幼児向け

### 制約・フォールバック

- 端末に日本語ボイスがない/オフラインでボイスが使えない環境がある
  (iOSはローカルボイスあり。Androidは端末による)。
  読み上げできない場合も**無音で従来どおり動く**こと(例外を握りつぶす)
- 読み上げの品質は端末依存。数の読み(「13こ」)が不自然な場合は
  読み上げ用テキストを別に持つ(例: 「ぜんぶで じゅうさんこ」)ことも検討

### 完了条件

- 出題・正誤メッセージが自動で読み上げられ、問題文タップで再読み上げできる
- 🔇 のとき読み上げない
- 日本語ボイスがない環境でもエラーなく従来どおり遊べる

## 2. ぴょこっとタッチ: 中断時のラウンド保護

### 課題

残り時間が `performance.now()` 基準のため、プレイ中に電話・画面ロック・アプリ切替が
あってもラウンドの時計は進み続ける。戻ってきたときにはラウンドが終わっていて、
消費した🎈チケットだけが失われる。1日3回制だからこそ1回の重みが大きい。

### 方針

タブが非表示になったらラウンドを一時停止し、戻ってきたら少しの猶予つきで再開する。

- `document.visibilitychange` で `hidden` になったら:
  - `state.running = false` にして `spawn` のタイマーと `tickTime` のrAFを止める
  - 残り時間(`state.endTime - performance.now()`)を保存する
  - 表示中のキャラは全部引っ込める(`hideChar`)
- `visible` に戻ったら:
  - 3-2-1カウントダウン(`runCountdown`)を挟んでから残り時間で再開する
    (いきなり再開すると子どもが構えられないため)

### 実装メモ

- 一時停止は「ラウンド進行中(`state.running`)」のときだけ。スタート画面・結果画面では何もしない
- カウントダウン中(`running` になる前)に隠れた場合は、復帰時にカウントダウンをやり直すだけでよい
- `endRound` 済みかどうかのガードに既存の `state.running` を使う

### 完了条件

- プレイ中にタブを裏に回して数十秒後に戻っても、残り時間が減っておらず、
  カウントダウン後に続きから遊べる
- スタート画面・結果画面・おやすみ画面では挙動が変わらない

## 3. チケット切れ時の「べんきょう」への導線

### 課題

🎈チケットが切れると、ぴょこのおやすみ画面(🌙)は🏠に戻るだけの行き止まり、
ハブも「きょうは おしまい 💤」と表示するだけ。
「あそびは1日ぶんまで・べんきょうは無制限」という設計思想があるのに、
遊びたい気持ちをべんきょうに流す導線がない。

### 方針

チケット切れの画面から、無制限で遊べる「✏️ べんきょう」へ誘導する。

- **ぴょこのおやすみ画面**: 🏠 の横に「✏️ べんきょうする」ボタンを追加し、
  `../../study/numakasten/` へ直接リンクする
  (将来べんきょうアプリが増えたらハブの✏️セクションへ誘導する形に変える)
- **ハブ**: 残り🎈が0のとき、あそびのタイルを薄く+💤を重ねて「押す前に分かる」ようにし、
  べんきょうセクションをさりげなく目立たせる(例: 「なんかいでも ⭕」バッジを強調)

### 実装メモ

- おやすみ画面のボタンは絵文字主体(✏️ + 🧮 など)にし、読めなくても意味が伝わるようにする
- ハブのタイル無効化は見た目だけにする(タップは従来どおりゲームへ→おやすみ画面。
  「押したら教えてくれる」経路も残す)
- 導線追加後も、あそびタイルからおやすみ画面に着地したときに同じ「べんきょうへ」ボタンが
  あることで一貫する

### 完了条件

- チケット0のとき、ぴょこのおやすみ画面からワンタップでヌマーカステンに移動できる
- ハブでチケット0が見た目で分かる
- チケットが残っているときの見た目・挙動は変わらない
